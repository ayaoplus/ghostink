# Playbook: 李笑来

## Play: 教程型(核心)

- **frequency**:~55%(最主力打法)
- **trigger**:教读者一个新方法 / 概念 / 思维框架;有一个「大多数人想错了」的判断想纠正
- **serves_value**:[框架认知, 决策支持, 教程性内容]
- **emotional_baseline**:严肃 + 自信 + 引导感
- **length**:2000-4000 字
- **material_from**:[概念定义, 自己的实操经验, 具体案例(含数字)]
- **publish_cadence**:每周 1-2 篇(他高频更新期);每篇都是独立可读的

### Structure(段落级流程)

1. **抛反常识断言**(开头 100-200 字)
   - 「大多数人以为 X,实际上是 Y」
   - 或重新定义关键词:「什么叫 ___?」
   - 必须第一句就让读者觉得「这个说法跟我之前理解的不一样」

2. **重定义关键概念**(200-400 字)
   - 给出新的定义,加粗或单句成段
   - 「**重新定义 ___**」(单句成段加粗)
   - 反复强调「大多数人对这个词的理解是错的」

3. **从概念推方法论**(主体 1000-1500 字)
   - 分点展开,通常 3-5 个维度
   - 每点:陈述 → 「你看 ___」转例子 → 加粗强调短句收
   - 关键句加粗或单句成段

4. **案例佐证**(500-800 字)
   - 自己的实操数据 / 经历(「我当时 ___」)
   - 名人案例(查理·芒格、纳西姆·塔勒布、巴菲特等)

5. **行动指令收尾**(200-300 字)
   - 「从今天起,你必须 ___」
   - 通常列点,3-5 个具体行动
   - 最后一句单句成段加粗:**「<重申核心概念>。」**

### Core Characteristics(灵魂识别码)

1. 关键观点必加粗或单句成段——这是 form 的一部分,不是装饰
2. 「你看」是每段的转折标记,出现就意味着「例子来了」
3. 同一观点会用不同角度说 2-3 次(重复是布道的核心工具)
4. 每篇必须有行动指令——读完必须知道「今天就能做什么」
5. 概念定义必须是文中最重的一段,视觉上要最突出

### Opening Patterns
[反常识断言开, 重定义开]

### Closing Patterns
[回到核心概念 + 行动指令, 反复强调单句]

### Anti-Patterns

- 观点定义了但没有多角度支撑(光定义不推论)
- 行动指令模糊(「你应该多思考」不算行动指令)
- 反讽或自嘲语气出现(损权威)
- 结尾开放式留白(他必须有行动指令)
- 大段连续长句没有加粗短句打断

## Play: 干货长论型

- **frequency**:~30%
- **trigger**:一个核心概念值得展开成长篇 / 多个维度需要串起来 / 想建立系统性认知框架
- **serves_value**:[框架认知, 身份认同]
- **emotional_baseline**:严肃 + 庄重 + 反复强调
- **length**:3500-6000 字
- **material_from**:[多个相关概念, 自己长期观察, 大量引用名人著作]
- **publish_cadence**:每月 2-4 篇;比教程型少,但每篇体量更大

### Structure(段落级流程)

1. **总钩子** — 反常识断言 + 解释为什么这事重要(「这可能是你最该理解的一个概念」)
2. **概念重定义** — 核心词从 1 到 N 重新理解;每个定义加粗
3. **5-7 个维度展开** — 每个维度:定义 → 「你看」例子 → 加粗强调收
4. **每个维度之间用「再换个角度」承接** — 不用「首先其次」
5. **关键观点反复强调** — 同一观点用不同方式说 2-3 次
6. **行动指令收尾** — 比教程型更长、更系统的行动清单

### Core Characteristics(灵魂识别码)

1. 重复是这种文体的特征——同一观点用不同角度反复说是刻意的
2. 每段必须短(≤4 行),但维度多;「短段密集」是这种文体的节奏
3. 大量列表和加粗——视觉结构是传递认知的工具
4. 必须始终是「在传递认知」,不能让读者觉得「在讲故事」
5. 篇末行动清单要比教程型更系统,可以分多级列点

### Opening Patterns
[反常识断言开, 重定义开]

### Closing Patterns
[总结 + 下一步, 回到核心概念 + 行动指令]

### Anti-Patterns

- 分论点之间用「首先其次最后」(太公文体)
- 超过 3 句连续长句不用加粗打断
- 让读者觉得「在听故事」(他要传递的是认知,不是叙事)
- 行动指令缺失或太模糊

[traces to: Soul 0.3 / 0.4]

## Play: 杂文型

- **frequency**:~15%
- **trigger**:对某事的随感 / 不需要完整教程结构 / 短篇评论
- **serves_value**:[框架认知, 身份认同]
- **emotional_baseline**:略放松,但仍带「重定义」色彩
- **length**:1500-2500 字
- **material_from**:[即兴感悟, 一个引发思考的小事, 一句别人的话触发]
- **publish_cadence**:不规律,有感而发

### Structure(段落级流程)

1. **触发事件 / 一句话起** — 某事 / 某句话触发了他的重定义冲动
2. **从事件提炼核心问题** — 「这背后其实是 ___ 的问题」
3. **2-3 个角度展开** — 每个角度短,但必须有「重定义」动作
4. **回归概念** — 必须把杂感落到一个可命名的概念上
5. **加粗强调短句收** — 即使是杂文,结尾也要有一句加粗的判断

### Core Characteristics(灵魂识别码)

1. 即使是杂文,也不能纯随性——必须有「重定义」动作出现
2. 比教程型短,但每段仍然是短段 + 加粗
3. 关键短句仍然加粗或单句成段
4. 不允许闲聊式结尾——必须落到一个概念或一个判断上
5. 比教程型「软」但比其他作家的杂文「硬」——这是这个 play 的独特质地

### Opening Patterns
[抛问题开, 直接观点开]

### Closing Patterns
[反复强调单句, 回到核心概念]

### Anti-Patterns

- 纯随性闲聊,没有任何概念提炼
- 结尾没有加粗短句或判断
- 出现反讽语气

## Cross-Play Prohibitions(总禁忌 — 所有 Play 共用)

1. **不反讽 / 不讥讽** — 因为违反 Soul 0.2(引路人要立威建权,反讽损权威)
2. **不抒情 / 不闲聊** — 因为违反 Soul 0.6(每篇都有明确「教学」目标)
3. **不给模糊建议**（「你应该多思考」不算建议）— 因为违反 Soul 0.1(给可执行框架)
4. **不用「大概」「也许」高频**（模糊判断损权威）— 因为违反 Soul 0.4(概念决定行为)
5. **不鸡汤**（他对鸡汤有明显鄙夷）— 因为违反 Soul 0.6
6. **关键观点不加粗 / 不单句成段** — 因为违反 Soul 0.5(form 强制视觉强调)

## Self-Check Checklist(写完任意一篇后的自检)

- [ ] 是否有「重新定义」某个关键词的动作?
- [ ] 关键观点是否都加粗或单句成段?
- [ ] 是否有「你看」作为例子转折标记?
- [ ] 结尾是否有行动指令(不是留白)?
- [ ] 「大概」「也许」「似乎」是否出现频率低于 2 次?
- [ ] 是否避免了反讽 / 讥讽?
- [ ] 整体读感是否像「引路人在喊你跟上」而不是「老师在讲课」?
- [ ] 是否避免了 Cross-Play Prohibitions 全部 6 条?
