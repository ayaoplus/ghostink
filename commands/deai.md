# /ghostink deai

任意中文文本去 AI 感扫描 + 修订建议。**只出报告,不阈值通过,决定权在你。**

> 这是 ghostink 的**独立一级命令**,不依赖任何作家的 form 或 soul。任意来源
> (你自己写的、AI 生成的、别人写的、混合的)中文文本都可以跑。

## 用法

```
/ghostink deai [file-path]              # 扫指定文件
/ghostink deai                          # 提示输入文本或选最近草稿
/ghostink deai [file] --with-form       # opt-in 加载 studio/form.md 调灵敏度
/ghostink deai [file] --apply           # 出报告 + 直接生成 _deai 修订版
```

## 设计哲学

四条不可妥协的原则:

1. **不阈值通过** — 不打 AI 度分数让你"过 60 分就发",任何分数下决定权都在你
2. **不依赖 form** — 默认通用文本扫描,跨任何作家 / 领域。`--with-form` 是 opt-in
3. **禁用词必配替换方案** — 任何"删 X"建议都要给"换成 Y"的具体替换,不能光说删
4. **活人感终审是主观判断,不机械化** — 4 个扫描层是机器能做的,第 5 层"活人感终审"必须以读者视角问一个核心问题,不打分

## 输入

- 待审查的中文文本(file-path 或 inline)
- (可选)`[studio]/form.md` — 加 `--with-form` 才加载;部分规则按 form factions 调整(反讽派 form 应用 Rule 2.4 时手感不同)

## 输出

- **结构化报告**(纯文本,按修复优先级排序)
- 不修原文件
- 加 `--apply` → 自动应用建议生成 `<file>_deai.md`(原文件不动)

---

## 五个扫描层 + 终审

按"机器能做 → 主观判断"递进。前 4 层都基于 50+ 条具体规则的模式匹配,
第 5 层"活人感终审"是以读者视角的整体判断。

```
层 1  零容忍清单(命中=必改,提供具体替换)
层 2  句式与结构模式(扫公式化结构)
层 3  词频与措辞模式(扫词级 AI 指纹)
层 4  内容深度模式(扫内容层 AI 痕迹)
─────────────────────────
层 5  活人感终审(主观,以读者视角)
```

## 层 2:句式与结构模式

### 2.1 句式重复(最隐形的指纹)

- AI 标志:相邻句结构相似(连续 3+ 句"主语 + 动词 + 宾语"同构)
- 检查:扫连续句子,标结构同构 ≥3 句
- 修复:打破节奏(短-长-中混排),或改一句为反问

### 2.2 结构可预测("太对"就是错)

- AI 标志:每段都"话题句 → 论据 → 结论";总分总僵硬
- 检查:
  - 段落开头是否都是话题句 → 标统一模式
  - 是否每段都有显式过渡词 → 标僵硬
- 修复:让某些观点中段冒出而非总开头;插一段看似离题再绕回;删显式过渡句

### 2.3 三段式法则过度

- AI 标志:强行把想法分成三组以显得全面("X、Y、Z 三件事";"无缝、直观和强大")
- 检查:连续 2+ 三项并列 → 标
- 修复:**改两项或四项**(两项优于三项);堆叠三个形容词改散文

### 2.4 否定式排比

- AI 标志:"不仅...而且";"这不仅仅是 X,而是 Y";"X 不只是 A,更是 B"
- 检查:全文搜以上模式
- 修复:拆成两个独立陈述,直说 Y
- **若 form.factions 含"反讽"**(opt-in):此规则灵敏度调低(王小波风格反讽排比应允许)

### 2.5 整齐分段(段落均长)

- AI 标志:每段长度差不多,视觉上"均匀"
- 检查:段长方差 → 标过低的(段长高度均匀,无短段呼吸点)
- 修复:加几个超短段(1-2 句)制造重量感和呼吸感

### 2.6 戏剧性分段(段落标题党)

- AI 标志:每段开头一个"小金句",后面是论述("黑暗森林。X 就是这种处境...")
- 检查:连续 3+ 段是这种"金句开头 + 论述"模式
- 修复:打破,让某段直接进论述

## 层 3:词频与措辞模式

### 3.1 万能词高频

- AI 标志:"非常 / 真的 / 这个 / 一些 / 可以 / 我们 / 这种" 高频
- 检查:扫高频词,标超过文本平均 3 倍的
- 修复:替换为更具体的词("很多" → "三十多个";"非常重要" → "决定生死")

### 3.2 同义词循环(刻意换词)

- AI 标志:同一对象用一连串同义词替换(主人公 / 主要角色 / 中心人物 / 英雄)
- 检查:扫指代同一对象的不同词,3+ 同义词标
- 修复:统一用一个词

### 3.3 系动词回避

- AI 标志:用"作为 / 代表 / 标志着 / 充当 [一个]"代替"是";用"拥有 / 设有 / 提供"代替"有"
- 检查:全文搜"作为...的 X" / "拥有 X" 模式
- 修复:换"是 / 有"
- 例:"Gallery 825 作为 LAAA 的当代艺术展览空间" → "Gallery 825 是 LAAA 的当代艺术展览空间"

### 3.4 -ing 肤浅分析(中文为"反映 / 强调 / 凸显..."从句)

- AI 标志:句末加"反映了 X / 象征着 Y / 凸显了 Z" 添加虚假深度
- 检查:扫"X,反映了 Y" / "X,象征着 Y" / "X,凸显了 Z" / "X,确保了 Y" 模式
- 修复:拆成两句独立陈述,或删第二句(肤浅)
- 例:"寺庙的蓝色与该地区的自然美景产生共鸣,象征着州花,反映了社区与土地的联系" → "寺庙用蓝色。建筑师说是为了呼应当地的州花。"

### 3.5 过度限定

- AI 标志:大量软化语("可能 / 应该 / 或许 / 一定程度上 / 潜在地 / 通常")
- 检查:每 100 字软化语 ≥ 3 个 → 标;"可以潜在地可能..." 这种叠加 → 严重 FAIL
- 修复:换具体判断或直接立场

### 3.6 模糊归因

- AI 标志:"专家认为""观察者指出""一些批评者认为""多个来源" — 实际未引用
- 检查:全文搜以上模式
- 修复:给具体名字 / 来源 / 出版日期;没具体来源就删该论断或承认是个人判断("我觉得")

### 3.7 虚假范围

- AI 标志:"从 X 到 Y" 但 X 和 Y 不在有意义的尺度上("从大爆炸到暗物质的神秘舞蹈")
- 检查:扫"从 X 到 Y" 结构,验 X 和 Y 是否同尺度
- 修复:直接列具体内容,删"从...到..."框架

### 3.8 全员视角("我们 / 作为人类")

- AI 标志:大量"我们""人们""作为 X 的一员",回避具体的"我"
- 检查:第一人称"我"频次 vs "我们"频次 → "我"过低标
- 修复:把"我们应该" → "我觉得";"作为程序员我们" → "我做程序员的"

### 3.9 填充短语

| 删 | 换 |
|---|---|
| 为了实现这一目标 | 为了 |
| 由于...的事实 | 因为 |
| 在这个时间点 | 现在 |
| 在 X 的情况下 | 如果 X |
| 系统具有处理...的能力 | 系统可以处理 |
| 值得注意的是数据显示 | 数据显示 |

## 层 4:内容深度模式

### 4.1 总结性收尾(教化 / 升华)

- AI 标志:文末写一段总结("这告诉我们 / 由此可见 / 总而言之 / 这就是为什么...")
- 检查:最后 1-2 段是否含总结 / 升华句 → 标
- 修复:删总结段,留一个具体细节作收(或反讽 / 闲话破节奏)

### 4.2 通用积极结论

- AI 标志:模糊的乐观结尾("公司的未来看起来光明""激动人心的时代即将到来""向正确方向迈出的重要一步""继续追求卓越的旅程")
- 检查:扫文末段,标乐观空话
- 修复:改具体计划 / 真实下一步 / 闭环到具体细节

### 4.3 提纲式"挑战与未来展望"

- AI 标志:文中插入"尽管面临 X 挑战...凭借 Y 优势继续蓬勃发展""挑战与遗产""未来展望"等标题段
- 检查:扫段落标题 + 内容模式
- 修复:删该段;有具体挑战就写具体的(用具体年份 / 具体项目)

### 4.4 过度强调意义 / 遗产 / 趋势

- AI 标志:"X 标志着 Y 的关键时刻""见证了 Z 的演变""不可磨灭的印记""为...奠定基础""不断演变的格局"
- 检查:扫宏大叙事词组(参见 1.1 词表)
- 修复:删宏大句,直接陈述事实

### 4.5 过度强调知名度 / 媒体报道

- AI 标志:"被 X、Y、Z 报道""活跃的社交媒体账号,X 万粉丝"(列堆名字不给上下文)
- 检查:扫媒体报道堆叠
- 修复:挑 1-2 个具体引用 + 内容("在 X 的采访中,她说 Y"),删其余堆叠

### 4.6 协作交流痕迹(AI 对话残留)

- AI 标志:"希望这对您有帮助""当然!""一定!""您想要..."
- 检查:扫感叹句 + 协作语气
- 修复:删

### 4.7 谄媚 / 卑躬屈膝

- AI 标志:"好问题!""您说得完全正确""这是一个复杂的话题"
- 修复:删

### 4.8 知识截止日期免责声明

- AI 标志:"截至 [日期] 我所知...""虽然具体细节有限...""根据可用信息..."
- 修复:删,或换具体来源 + 日期

### 4.9 标题大写化(中文罕见但中英混排时出现)

- AI 标志:中英混排标题,英文部分把每个主要单词都首字母大写("Strategic Negotiation And Global Partnerships")
- 修复:改 sentence case("Strategic negotiation and global partnerships")

## 层 5:活人感终审(主观,以读者视角)

**这是最重要也最主观的一层**。不是逐项扫描,而是以读者视角通读全文,问一个核心问题:

> **"读完这篇文章,我感觉是一个**有立场、有具体经验、有点小毛刺**的真人在
> 跟我聊一件他真的关心的事,还是一个 AI 在给我输出'信息'?"**

### 4 个感知维度(配套问题,主观判断)

#### 5.1 温度感

- 文中的情绪表达是**体感记忆**还是**知识性描述**?
- 体感:"我当时就愣住了" / "凌晨三点没人看着的时候,智能体还在不停地运转,这让人不安"
- 知识性:"我感到非常震撼" / "这令人担忧"
- 不通过修复:把每处"感到 X"换成具体场景细节

#### 5.2 独特性

- 这篇文章是否有"换一个 AI 博主也能写出差不多"的感觉?
- 缺独特性的标志:观点都是"行业普遍认为"那种;没有"只有这位作者会写出来的角度"
- 不通过修复:加 1-2 处反主流观点,带个人经历佐证

#### 5.3 姿态

- 文章语气是不是"一个有立场的普通人在聊一件打动他的事"?
- 偏移信号:"导师在教学生"姿态(满篇"你应该""你必须");"品牌在做营销"姿态(满篇形容词);"维基在写词条"姿态(满篇中立陈述)
- 不通过修复:把"X 应该 Y" → "我觉得 X 该 Y";把营销词换成具体产品细节;把中立陈述加上"我"的视角

#### 5.4 心流

- 从头到尾读,有没有哪个地方注意力断掉、需要回头理解逻辑?
- 那个地方就是节奏问题(段太长 / 跳跃太大 / 套话过多)
- 不通过修复:锁定断点段重写

## 与 form 的 opt-in 联动

加 `--with-form` 时:

1. 解析 `[studio]/form.md`(若不存在 → 提示)
2. 加载 `factions` 字段
3. 调整规则灵敏度:

| form.factions 含 | 调整 |
|---|---|
| 反讽 | 层 2.4 否定式排比灵敏度 -50%(反讽派合理用否定排比);层 1.1 部分宏大叙事词允许(反讽时有用) |
| 长句分析 | 层 2.1 句式重复灵敏度 -30%(长句分析自然有结构相似) |
| 短句轰炸 | 层 2.5 整齐分段灵敏度 -50%(短句轰炸天然段落不均) |
| 抒情 | 层 4.4 过度强调意义灵敏度 -30%(抒情允许部分宏大) |

## 调用模式

### 模式 A:独立扫任意文本

```
/ghostink deai some-article.md
```

按上述五层扫,出报告,等用户决定。

### 模式 B:write 流程内部调用(Step 5)

`commands/write.md` Step 5 自动调本命令:
- 不需用户手动跑
- 自动加载 studio/form.md(--with-form 默认 ON)
- 报告呈现给用户后询问"全部修 / 选择性修 / 跳过"

### 模式 C:批量扫多个文件

```
/ghostink deai drafts/*.md
```

逐文件扫,每文件出独立报告,最后给汇总。

## 实现备注

- **不阈值通过** — 不要"75 分以下不能发"这种逻辑。任何分数下决定权在用户
- **禁用词必配替换** — 1.1 表里"删"以外的每条都给替换;只列禁用词不给替换的规则不是合格规则
- **不修原文件** — 任何修订写新文件加 `_deai` 后缀
- **报告语气保持中性** — 不要"惩罚式"措辞("这写得很 AI" / "这段有问题");用"扫到 X / 建议 Y"
- **--apply 必须二次确认** — 即使加了 --apply,生成 _deai 文件前要 echo 一次"将应用 N 条修改,确认?"
- **中文输出**

## 这个命令跟 ghostink 其他命令的关系

| 命令 | 角度 |
|---|---|
| `/ghostink check [file]` | 检查文笔是否符合**当前作家 form**(form-check 14 节验证)— 必须有 form |
| `/ghostink deai [file]` | 检查文本是否有 AI 痕迹(50+ 通用规则)— 不需要 form,通用 |
| `/ghostink write` Step 4 | 内部跑 form-check |
| `/ghostink write` Step 5 | 内部跑 deai(with --with-form ON) |

**check 看"像不像猫笔刀",deai 看"像不像 AI"** — 两者互补,都跑能更稳。
