# _internal/brainstorm

把粗糙想法变成结构化骨架(skeleton)。被 `commands/write.md` Step 2 调用。

## 输入

- topic(从 write 命令行)
- `[studio]/soul.md`(挑战框架的依据)
- chosen_play(write Step 1 已选定)
- `[studio]/playbook.md`(取 chosen_play 的结构模板)
- `[studio]/profile/experiences.md` / `opinions.md`(选素材)
- `[studio]/profile/identity.md`(读者画像)
- material(可选,用户提供的素材文档夹)

## 输出

`[studio]/drafts/_brainstorm/YYYY-MM-DD-<slug>.md`,frontmatter + 5 节:

```yaml
---
date: YYYY-MM-DD
topic: <主题>
play: <chosen_play>
status: brainstormed
---

## 核心判断
## 差异化切入
## 可用素材
## 大纲
## 对话摘要
```

## 核心原则

### 动态问题原则(每个问题必须满足)

- 引用用户之前说过的具体词句
- 答案是 yes/no、A-or-B 或从短列表选
- 配 3-4 个具体候选(用 AskUserQuestion)
- 含一行意图说明:"我问这个是为了帮你 X"

### 禁止的问题模式

- 开放式元问题:"你想表达什么?""你的核心观点是什么?"
- 抽象受众探问:"你的目标读者是谁?"(已经在 profile 里有了)
- 假谦让:"你怎么想?"
- 用户用"我不知道,所以才来"就能回答的问题

### 收敛规则

每步有"目标产物",拿到就走;3 轮拿不到就收尾,不要无限轮回。

### 用户随时可中断

```
任何时候你可以说"跳过这步""直接出大纲""换个方向",我会立即收敛。
```

## Step 1 — 主题分流

接到 topic 后,判断三种情况:

### Branch A:清晰主题 + 清晰立场
(如"AI 不会取代初级程序员,反而让他们更值钱,我想用带实习生经验作素材")

→ 跳过 Step 2-4,直接进 Step 5(素材对齐)和 Step 6(出大纲)

### Branch B:有主题但立场模糊
(如"我想写写最近 AI 编程工具的事")

→ 进 Step 2

### Branch C:"不知道写啥"

扫 `profile/experiences.md` 和 `opinions.md`,找出 3-4 个候选话题(经历 + 观点配对)。

```
从你 profile 里挖出几个方向:
- A) opinions.md 里你写的 [观点 X] + experiences.md 的 [经历 Y]
- B) ...
- C) 都不对,我说说这周让我有感而发的事
```

用户选了 → 当 Branch B 处理。

profile 为空时:坦白说"profile 里没料,讲讲你最近什么事让你想写"。

## Step 2 — 锐化判断(目标:一句具体到能被反驳的话)

### 必须

- 引用用户已经说过的词
- 给 3-4 个候选(从安全 → 锋利)
- 状意图:"我问这个是想帮你落到一个具体到能被反驳的判断"

### 例

```
你这话题"DeepSeek 开源",你最想说什么?
- A) 开源是好事,证明中国 AI 能力(太平,所有人都在说)
- B) 开源不重要,成本结构才是冲击
- C) 美股估值逻辑被打穿,这是金融事件不是技术事件
- D) 都不对(自由输入)

我问这个是为了找到一个别人没大声说的判断 — A 太平,BC 可能更接近你的角度。
```

3 轮拿不到具体判断 → "聊三轮还是泛,要不跳过这步,从素材倒推?" → 进 Step 5。

## Step 2.5 — 挑战框架(单次,可跳)

加载 soul.md。判断:

1. 用户判断是否真的反主流?还是平庸披了反话外衣?
2. 用户是否说了什么和判断矛盾的话?
3. 选的 play 是否匹配情绪重量?

**有可挑战的**:发一次 reframe(必须引用用户原话):

```
打断一下 — 你刚才说"<引用原话>"。这话比你选的 [B] 更有意思。
我怀疑你真正想写的不是 <他们说的主题>,而是 <重新框定>。

- A) 对,换这个框架
- B) 不对,保持原来的
- C) 部分对,我调整一下(自由输入)

我问这个是因为 <为什么这个框定更锋利>。
```

**没有合理挑战**:坦白说"你的判断已经够锋利,不需要挑战",跳到 Step 3。

绝不为了挑战而挑战。用户拒绝时立即放弃 reframe。

## Step 3 — 差异化角度(目标:至少一个独特切入)

提示词:
- "这判断 [知名作者/媒体] 也在讲。你和他们差别是什么?"
- "如果读者已经同意你的判断,你还想告诉他们什么他们不知道的?"
- "这话题里最少被提到、但你觉得关键的一点是什么?"

每个问题给 3-4 候选(从 profile/experiences、opinions、soul 0.4 信念里挖)。

最多 2 轮。

## Step 4 — 素材对齐(目标:1-3 条具体素材)

扫 `experiences.md` 找标签或关键词与主题重合的:

```
profile 里有这些相关素材,用哪些?
- A) E012 [带实习生那次] — "初级程序员"贴合
- B) E024 [自己第一份工作犯的低级错] — 可以做类比
- C) E031 [看到 AI 工具上手门槛] — 贴判断
- D) 都不合适,我有别的素材

我问这个是为了让文章有真实细节,不写成"AI 分析报告"。
```

profile 为空 → 直接让用户提供 1-3 个素材。

选完后悄悄检查:素材是否真的支持判断?不一致就提醒:

```
我注意到 E024 是低级错误,但你判断是"AI 让初级程序员更值钱"。
这两个方向有点拧。换素材还是调判断?
```

## Step 5 — 出大纲

读 chosen_play 的 structure,套用:
- 按 play structure 的步骤一步步填
- 每步 1-2 句描述
- 标明用了哪条素材
- (可选)字数估计

直接出,不要走对话。一次给完,问"满意吗?要改哪里?"

最多 3 轮修订。

## Step 6 — 写 artifact

格式:

```markdown
---
date: YYYY-MM-DD
topic: <用户原 topic>
play: <chosen_play>
status: brainstormed
---

## 核心判断
<one-sentence>

## 差异化切入
<1-3 句>

## 可用素材
- [E012 带实习生] — <一行用法>
- [E031 AI 工具] — <一行>
- [新增: xxx] — <一行>

## 大纲
<按 play 模板填>

## 对话摘要
<3-5 bullets:判断怎么锐化的、有没有挑战、为什么这个角度赢了>
```

写到 `[studio]/drafts/_brainstorm/YYYY-MM-DD-<slug>.md`。

slug = 主题取前 4-6 个有意义词,kebab-case 或 pinyin。

返回给 write 命令(write Step 2 收到这个文件后进 Step 3 draft)。

## 实现备注

- 总会话长度 8-15 条消息,不要 40 条
- 每个 AskUserQuestion 是真工具调用,不是 markdown 文本
- 步与步之间不要重复总结
- 中文叙述,关键术语保留英文
