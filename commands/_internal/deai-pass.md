# _internal/deai-pass

去 AI 感扫描。**只出报告 + 建议,不阈值通过,决定权在用户。**

被以下两处调用:
- `commands/write.md` Step 5(自动审 form-check 后的草稿)
- `commands/deai.md`(用户用 `/ghostink deai [file]` 处理任意文本)

## 核心理念

**AI 感不是基调问题,是可预测性问题。** 当读者读完第一段已经能猜到下一段写什么,信任就丢了。

## 输入

- 待审查的文本
- (可选)`[studio]/form.md`——若已存在,某些规则按 form 调整(如反讽派 form 应用 Rule 5 时手感不同)

## 输出

结构化报告(纯文本)。**不修改原文件。**

## 9 条规则

### Rule 1:标点审计 — 第一过滤器

- AI 标志:每句都句号收尾;引号成对出现;破折号造作停顿
- 检查:
  - 句号 vs 逗号比例 → 句号 ≥ 逗号 标 AI-heavy
  - 装饰性引号(非真引用、非反讽用)→ 列每处
  - 破折号集群(3+ 在近距离)→ 标
  - 完全无终止标点过渡的句子(自然流入下句)→ 0 处则建议加几处
- 修复方向:部分句号转逗号、删装饰引号、破折号换逗号或换行

### Rule 2:结构可预测 — "太对"就是错

- AI 标志:话题句 → 论据 → 结论(每段如此);"首先...其次...最后"
- 检查:
  - "首先/其次/然后/最后" → 直接标
  - 每段是否都是话题句开头 → 标统一模式
  - 是否总分总结构 → 标僵硬
- 修复:让某些观点中段冒出而非总开头;插一段看似离题再绕回;删显式过渡句

### Rule 3:句式重复 — 最隐形的指纹

- AI 标志:相邻句结构相似(如都是"主语 + 动词 + 宾语")
- 检查:
  - 连续 3+ 句结构同构 → 标
  - 段落开头句式重复 → 标
- 修复:打破节奏(短-长-中混排)

### Rule 4:词频集中 — 万能词

- AI 标志:"非常""真的""可以""我们""这个""一些"高频
- 检查:扫高频词,标超过文本平均 3 倍的
- 修复:替换为更具体的词(把"很多"改 "三十多个")

### Rule 5:对仗与排比 — 写作上的"美甲"

- AI 标志:四字成语堆叠,排比"我们必须 X,必须 Y,必须 Z"
- 检查:
  - 连续 2+ 四字成语 → 标
  - 三句排比结构 → 标
- 修复:打散对仗,长短交错
- **若 form.factions 含"反讽"或"长句分析"**:此规则灵敏度调低(王小波风格的修辞排比应被允许)

### Rule 6:抽象空话 — "应该""可能""一种"

- AI 标志:大量软化语("可能""应该""或许""一定程度上")
- 检查:每 100 字软化语 ≥ 3 个 → 标
- 修复:换具体判断或直接立场

### Rule 7:总结性收尾 — 教化与升华

- AI 标志:文末写一段总结("这告诉我们/由此可见/总而言之")
- 检查:最后 1-2 段是否含总结/升华句 → 标
- 修复:删总结段,留一个具体细节作收

### Rule 8:整齐分段 — 段落均长

- AI 标志:每段长度差不多
- 检查:段长方差 → 标过低的(段长高度均匀)
- 修复:加几个超短段(1-2 句)制造呼吸感

### Rule 9:全员视角 — "我们""作为人类"

- AI 标志:大量"我们""人们""作为 X 的一员",回避具体的"我"
- 检查:第一人称"我"频次 vs "我们"频次 → "我"过低标
- 修复:把"我们应该"改 "我觉得"

## 报告格式

```
Ghostink De-AI Pass
====================
Target: <filename / inline>
Form context: <form name 或 N/A>

Overall: <分数,参考用,不阈值通过>

RULE 1 标点审计      [PASS / WARN / FAIL]
  - 句号 vs 逗号: 23 vs 12 (1.9x) — WARN
  - 装饰引号: 3 处 (行 5, 12, 28)
  建议:把行 5 的引号去掉(非真引用)

RULE 2 结构可预测    [PASS]
  - 无显式过渡词
  - 段落开头多样

RULE 3 句式重复      [WARN]
  - 行 8-10 三句"主-动-宾"同构
  建议:把第 9 行改长句或反问句

...

OVERALL SUGGESTIONS (按优先级):
1. <最影响的修改>
2. <...>
3. <...>
```

## 报告后处理

### 被 write Step 5 调用时

write 询问用户:

> 你想怎么处理?
> - A) 全部修(自动应用建议)
> - B) 选择性修
> - C) 跳过,这版我接受(包括 AI 感我决定保留)

### 被 deai 命令独立调用时

仅显示报告,询问:

> 要应用建议生成修订版吗?(原文件不动,新文件加 _deai 后缀)

## 实现备注

- **不阈值通过**——70 分通过、85 分自动跳的逻辑已废弃。任何分数下,决定权都在用户
- 与 form.md 联动:某些规则在反讽派 / 长句分析派 form 下灵敏度调低
- 报告语气保持中性,不要"惩罚式"措辞
- 输出中文
